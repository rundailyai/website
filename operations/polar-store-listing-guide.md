# Polar Store Product Listing Guide

## Overview
This guide outlines the complete process for listing a digital product on our Polar store at rundaily.ai.

## Prerequisites
- Product files ready (PDF, ZIP, or other digital format)
- Product description and marketing copy finalized
- Pricing determined
- Preview images/screenshots prepared
- Polar account access

## Step-by-Step Process

### 1. Product Preparation
- [ ] Finalize product content and verify quality
- [ ] Create product thumbnail (recommended: 1200x630px)
- [ ] Create additional preview images (2-4 images recommended)
- [ ] Write product title (max 60 characters for SEO)
- [ ] Write short description (1-2 sentences, 150 characters)
- [ ] Write full product description (include benefits, what's included, who it's for)
- [ ] Define product category/tags

### 2. Polar Dashboard Setup
- [ ] Log into Polar dashboard at polar.sh
- [ ] Navigate to Products section
- [ ] Click "Create Product"
- [ ] Select product type: "Digital Product"

### 3. Product Information
- [ ] Upload product files
  - Maximum file size: 500MB
  - Supported formats: PDF, ZIP, MP4, etc.
  - Ensure files are named clearly
- [ ] Add product title
- [ ] Add short description
- [ ] Add full description (supports Markdown)
- [ ] Upload thumbnail image
- [ ] Upload preview images

### 4. Pricing & Access
- [ ] Set product price (in USD)
- [ ] Choose pricing model:
  - One-time purchase
  - Subscription (if applicable)
- [ ] Set download limits (recommended: unlimited)
- [ ] Configure access period (recommended: lifetime)

### 5. SEO & Marketing
- [ ] Add SEO meta description
- [ ] Add relevant tags/keywords
- [ ] Set product URL slug
- [ ] Add FAQ section (optional but recommended)

### 6. Testing
- [ ] Save as draft
- [ ] Preview product page
- [ ] Test download/access functionality
- [ ] Review on mobile device
- [ ] Check all links and descriptions

### 7. Launch
- [ ] Set product status to "Published"
- [ ] Verify product appears on store page
- [ ] Test purchase flow (use test mode if available)
- [ ] Share product link on social media
- [ ] Add to email newsletter (if applicable)

### 8. Post-Launch
- [ ] Monitor initial sales/downloads
- [ ] Respond to customer questions within 24 hours
- [ ] Collect and review customer feedback
- [ ] Update product description based on feedback

## Common Issues & Solutions

**Issue**: Product file won't upload
- Solution: Check file size (max 500MB), try compressing or splitting files

**Issue**: Preview images not displaying correctly
- Solution: Ensure images are in JPG/PNG format, check dimensions match requirements

**Issue**: Product not appearing on store
- Solution: Verify product status is "Published", check visibility settings

## Best Practices

1. **Clear Descriptions**: Be specific about what customers get
2. **Visual Assets**: Use high-quality images that showcase the product
3. **Pricing**: Research competitor pricing, consider introductory offers
4. **Updates**: Keep product files updated, notify customers of major updates
5. **Support**: Respond to questions quickly to maintain good reviews
